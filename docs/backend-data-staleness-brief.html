Why the UI shows 26 May 2026

Backend handoff · MindWealth Alpha Terminal · 10 June 2026

TL;DR: The frontend is connected to the live API. It is not using mock data. Signal/conviction/analytics data is stuck at 2026-05-26 because that is the newest date in conviction_store / trade_store/US/. The daily conviction pipeline has not published overlays after 26 May.

1. Frontend status (not a UI bug)

  • UI server calls NUXT_API_BASE_URL → http://51.20.53.218:8506 (API v1.2.0).
  • GET /api/meta returns live label: MindWealth API v1.2.0.
  • Mock fallbacks use 12 May or 17 May — seeing 26 May proves live API, not mocks.
  • “Last updated” in the topbar is parsed from the overlay CSV filename (e.g. 2026-05-26_outstanding_signal.csv).

2. What the backend returns today

Overlay dates currently exposed: ["2026-05-15","2026-05-20","2026-05-25","2026-05-26"] — no June dates exist yet.

2. Data flow (simplified)

Daily pipeline (backend)  →  trade_store/US/YYYY-MM-DD_*.csv
                         →  conviction_store overlays
                         →  GET /api/v1/... (MindWealth API)
                         →  Nuxt /api/* routes (MindWealth UI)
                         →  Browser

The UI always requests the latest overlay when no report_date is passed. It cannot show 10 June data until the backend creates 2026-06-10_* (or latest trading day) files.

3. Root cause

POST /api/v1/conviction/pipeline/daily (run_daily_conviction_pipeline) appears not to have run successfully — or not on schedule — since 26 May 2026. Without new overlays, every CSV-backed screen (dashboard KPIs, signals, conviction, performance, breadth, sentiment, shortlist) will keep showing 26 May.
