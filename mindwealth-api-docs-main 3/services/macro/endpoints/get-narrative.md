# GET /api/v1/macro/narrative

## Summary

Latest nightly brief — Claude narrative, system recommendation, regime snapshot, dominant signal and reason.

## HTTP

- **Method:** `GET`
- **Path:** `/api/v1/macro/narrative`
- **operationId:** `get_nightly_narrative`
- **Status:** implemented (v1.3.0)
- **Typical response:** 200, 404

## Maps to

`api/services/macro_service.get_narrative()` → reads `runic_output.json`

## Response shape

```json
{
  "date": "2026-06-18",
  "narrative": "Tactical tight money with strategic easy money backdrop. Combo C wk 11 MEDIUM ...",
  "system_recommendation": "Hold core longs (F wk 8). No new longs (D watch). C MEDIUM active.",
  "brave_fearful_display": "TACTICAL TIGHT MONEY / STRATEGIC EASY MONEY",
  "dominant_signal": "C",
  "dominant_reason": "Combo C active (week 11, MEDIUM). 83% 3M hit rate.",
  "regime": {
    "fed_cycle": "CUTTING_EARLY",
    "curve_regime": "STEEPENING",
    "geo_overlay": "REGIONAL_WAR",
    "val_regime": "EXTREME",
    "liquidity": "GLOBAL_EASY"
  },
  "cftc_status": "PENDING_3DAY_LAG"
}
```

## Example

```bash
curl -s http://51.20.53.218:8506/api/v1/macro/narrative | jq '.narrative'
```

## Notes

- `narrative` is generated by Claude Sonnet 4.6 API during the nightly run. It will be an empty string if the last run used `use_claude=False`.
- `system_recommendation` is Python-generated (deterministic, no LLM dependency).
- Mapped to the "NIGHTLY BRIEF" tab in the frontend.
- For the full nightly JSON including all section data, use `GET /macro/runic/nightly`.
